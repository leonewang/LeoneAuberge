问题粉碎机：Node.js

2015年8月13日

问题描述 node.js 创建基本 HTTP 服务器，request 请求执行多次。

详细描述 代码如下：

var http = require('http');
var count = 0;
http.createServer(function (req, res) {
console.log('Running...');

res.writeHead(200, {'Content-Type': 'text/plain'});
res.end('Hello World');
}).listen(1337, '127.0.0.1');
console.log('Server running at http://127.0.0.1:1337/');

// 控制台输出
// Running...
// Running...
`</pre>

<mark>解决方案</mark> 控制台输出表明内部方法被执行了两次，原因可能是 `request` 请求了两次，于是输出**请求路径**查看。

<pre>`var http = require('http');
var count = 0;
http.createServer(function (req, res) {
count ++;
console.log('Runtime: ' + count + '. Request: ' + req.url);

res.writeHead(200, {'Content-Type': 'text/plain'});
res.end('Hello World');
}).listen(1337, '127.0.0.1');
console.log('Server running at http://127.0.0.1:1337/');

// 控制台输出
// Runtime: 1\. Request: /
// Runtime: 2\. Request: /favicon.ico

所以原因是自身页面请求一次，而 /favicon.ico 获取站点图标也是一次请求，问题解决。

而将 req.end() 方法删除又会发生奇怪的事，请求数量又变为 1 ，并且 /favicon.ico 不发出请求，所有请求都变为 padding 状态。
