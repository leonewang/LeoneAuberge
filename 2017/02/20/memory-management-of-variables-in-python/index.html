Memory Management of Variables in Python

Feb 20, 2017

This article will not only expose the basic memory management in Python, but some mistaken tricks of passing argument to function.

Memory management of variables

There are variety of programming languages in which have quite different memory assignment strategies. When we do an assignment in C:

Let’s imagine the following situation: we put 1 into a bottle which is created with name variable a and used as a container. This statement creates a block of memory space so that it can hold the value for variables.

If want to change the value in the bottle, just put another value in instead of the old one as the bottle will be updated.

Assigning one variable to another will create a new block of memory and make a copy of the value, then put that into the new bottle.

Memory management in Python is much more weird since variables work more like tags. When do assignment in Python:

Python will put a tag named a bound to the value 2.

As you can imagine, tag a will be changed to the new value 2 in memory.

The statement can make a new tag called b bound to the same value 2. Moreover, no worries about the former value 1, this kind of housekeeping job of freeing unuse memory will be done by Python’s Automatic Garbage Collection.

Mutable assignment and passing argument of function

There are two different means to pass argument to function:

  • Call-by-value: in call-by-value, the argument expression is evaluated and result of this evaluation is bound to the corresponding variable in the function. So if the expression is a variable, a local copy of its value will be used. The variable in caller’s scope will be never changed when the function returns.

  • Call-by-reference: A function gets a implicit reference to the argument, rather than a copy of its value. As a consequence, the argument can be modified as variables can be accidently changed in a function call. However, this mean has the characteristics: greater time and space-efficiency because it do not need to copy the value of variables.

Note: Python uses call-by-reference by default.
